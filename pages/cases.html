⚙️ Backoffice Requests

Что было: Обработка заявок выполнялась вручную через корпоративную почту. Не было единой платформы для исключения дубликатов, сотрудники не имели точной информации о текущем статусе заявки. Адреса указывались по-разному, отсутствовало разделение по типам задач, все проверки выполнялись вручную.

Что сделано:

  • Разработан и внедрён workflow для всех типов заявок (QR/POS, E-Com, m-Com, расторжение договоров, тарифы, реквизиты и др.).
  • Настроены динамические формы (Extension for Jira Service Desk).
  • Созданы Groovy-скрипты для:
    • автосоздания связанных задач в проектах GTE, KKM, BANKPOS, DCE,
    • копирования определённого вложения (среди 10+ файлов) в другой проект через ScriptRunner Jobs с валидацией по наименованию,
    • назначения согласующих и автоматической отметки "approved".
  • Добавлены валидации: проверка ИНН, адреса (поле мультивыбор), согласующих лиц.
  • Настроены кастомные поля, схемы типов задач и схемы экранов.
  • Настроены экраны переходов — при смене статуса открываются обязательные для заполнения поля.
  • Реализована в Jira Service Desk панель «Заявки по этому ИНН» для быстрого поиска и отображения всех связанных задач.

Результат:

  • Устранено дублирование заявок.
  • Повысилась прозрачность процессов — виден текущий статус на каждом этапе.
  • Автоматизированы проверки и согласование.
  • Скорость обработки заявок выросла в 3 раза, количество ошибок снизилось на 40%.

⏰ ScriptRunner Jobs — Автоотчёты

Что было: Отчёты формировались вручную, данные о мерчантах передавались по почте без автоматизации. Ежемесячные выгрузки по проектам GTE и DCE собирались вручную и занимали значительное время.

Что сделано:

  • Реализованы 5 автоматических Jobs в ScriptRunner на Groovy:
  • Отправка ЛК O!Market — ежедневно в 18:00 собирает задачи типа «Заявки E-Com» и «Заявки m-Com» со статусом «Готово», формирует письмо с логинами/паролями/SID мерчантов и отправляет на почту. Метка lk_sent защищает от дублей.
  • KPI отчёт — 1-го числа каждого месяца в 10:00 выгружает задачи за прошлый месяц в CSV и отправляет на почту.
  • Отчёт по обновлениям анкет — автоматическая рассылка при изменении анкетных данных.
  • Отчёт по QR/POS — плановая выгрузка статистики по терминалам.
  • Отчёты по GTE и DCE — 2-го числа каждого месяца в 10:00 формирует два CSV-файла с задачами за прошлый месяц (автор, статус, поставщик и др.) и отправляет единым письмом с вложениями.

Результат:

  • Полностью устранена ручная работа по формированию и рассылке отчётов.
  • Данные о мерчантах доставляются автоматически каждый день без участия сотрудников.
  • Ежемесячные отчёты формируются и отправляются по расписанию.
